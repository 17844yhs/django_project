6.default 是 Python 层面的默认值，用来在 Django 的模型实例中设置默认值。
db_default 是数据库层面的默认值，用来设置数据库表字段的默认值。
在绝大多数情况下，default 足够使用，因为它已经在 Python 和数据库层面都起到了作用。但是如果你需要在数据库层面确保默认值的存在，尤其是当你直接操作数据库（例如通过 SQL）时，使用 db_default 会更合适

7.假设有一个 Author 和 Book 的模型关系，如果我们使用：
author = models.ForeignKey(Author, on_delete=models.CASCADE, related_name='books', related_query_name='book')
related_name='books'：在反向查询时，Author 实例通过 books 访问其相关的 Book 实例。
related_query_name='book'：在查询时，你可以通过 book 来进行查询过滤，比如：Author.objects.filter(book__title='Some Title')。

8.问题1：max_quantity = serializers.SerializerMethodField(read_only=True) 对应哪个方法？
在 DRF 序列化器中，SerializerMethodField 需要配套一个对应的 get_字段名 方法。这里：
max_quantity = serializers.SerializerMethodField(read_only=True)
对应的是：
def get_max_quantity(self, obj):
    """返回当前商品的最大可购买数量（库存）"""
    return obj.goods.stock
这是 DRF 的约定规则 - 当使用 SerializerMethodField 时，会自动查找名为 get_<field_name> 的方法来获取值。

问题2：为什么模型已经有 subtotal 属性还要在序列化器中定义 get_subtotal？
1. 显式控制序列化行为
DRF 不会自动识别模型的 @property 作为可序列化字段。
2.未来扩展灵活性
如果以后需要修改计算逻辑（比如添加折扣），只需改序列化器方法而不影响模型

11.序列化器验证库存数量和模型验证库存数量
验证层级	触发场景	优点	缺点
模型层	调用 save() 时触发	确保最终数据一致性	错误反馈不够详细
序列化器层	调用 is_valid() 时触发	可提前返回友好错误信息	需要与模型层保持同步

14.attrs 是自己创建的新字典吗？
        attrs 是 Django 序列化器在执行 validate 方法时传递的一个字典。它包含了所有字段的值，并可以在此字典上进行修改或验证。你可以理解为 attrs 是一个临时字典，它包含了客户端发送的所有数据字段。
        在方法内修改 attrs 可以影响后续的验证和数据保存。你可以在 attrs 中加入额外的字段，或者根据业务逻辑进行修改。
为什么要返回 attrs？
        在序列化器的 validate 方法中，返回 attrs 是为了确保经过验证后的数据被正确地传递到下一步（例如保存数据到数据库）。如果在验证过程中做了修改或添加额外的处理，返回 attrs 以便后续的步骤可以使用这些修改后的数据。
# 视图调用
serializer = CartItemWriteSerializer(data=request.data)
# 序列化器内部
self.instance → None  # 因为是新增，没有已存在的实例
attrs → {'goods': <Goods>, 'quantity': 2, 'user_phone': '13800138000'}  # 来自请求的新数据

# 视图调用
item = CartItem.objects.get(id=1)
serializer = CartItemWriteSerializer(instance=item, data=request.data)
# 序列化器内部
self.instance → <CartItem id=1>  # 已存在的购物车商品实例
attrs → {'quantity': 3}          # 用户想修改的新数据

16.方法	调用场景	参数来源
create()	当调用 serializer.save() 且序列化器没有实例（即新增操作）时	validated_data 来自 serializer.is_valid() 后的验证数据
update()	当调用 serializer.save() 且序列化器有实例（即修改操作）时	instance 是当前修改的对象，validated_data 来自验证后的数据

# 当你在视图调用 serializer.save() 时：
serializer.save()  
# 等价于：
serializer.create(validated_data=serializer.validated_data)
validated_data 包含：
        通过 serializer.is_valid() 验证后的数据
        从请求数据中提取的字段（如 goods_id, quantity）
        你在 validate() 方法中手动添加的数据（如 cart 对象）

# 当你在视图调用 serializer.save() 且序列化器已绑定实例时：
serializer.save()  
# 等价于：
serializer.update(instance=serializer.instance, validated_data=serializer.validated_data)
instance：视图中通过 get_object() 获取的模型实例
validated_data：验证后的请求数据

在 更新操作（PUT/PATCH）时，DRF 会自动将数据库中的现有对象传入序列化器作为 instance
在 创建操作（POST）时，self.instance 为 None

19.args：
args 是一个元组，通常用来获取通过 位置参数 传递的数据。
在 Django 中，args 一般用于获取通过查询字符串（例如 GET 请求中的参数）传递的内容或者函数调用中的位置参数。

kwargs：
kwargs 是一个字典，表示 关键字参数，通常用于获取通过 URL 路由传递的 动态路径参数（例如 URL 中指定的 pk 或 user_id）。
在 Django 的视图中，URL 路由模式会根据路径中的动态部分自动将它们作为关键字参数传递给视图函数（或方法）。例如，在一个 URL 模式为 path('user/<int:pk>/', views.UserDetail.as_view()) 中，pk 就会作为关键字参数传递给 kwargs。
