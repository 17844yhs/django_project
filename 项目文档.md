# 小米商城项目文档

## 1. 项目概述

小米商城是一个基于前后端分离架构的电商平台，主要功能包括用户登录注册、商品浏览、购物车管理、地址管理等。

### 1.1 核心功能
- 用户认证：登录、注册、验证码获取
- 商品管理：商品分类、商品列表、商品搜索
- 购物车管理：添加商品、更新数量、删除商品
- 地址管理：获取地址列表、添加新地址
- 轮播图展示：首页轮播图

## 2. 技术栈

### 2.1 后端技术
- **框架**：Django 5.2.4
- **API框架**：Django REST Framework
- **数据库**：MySQL
- **认证方式**：JWT
- **跨域处理**：django-cors-headers

### 2.2 前端技术
- **框架**：Vue 3
- **状态管理**：Pinia
- **路由**：Vue Router
- **HTTP客户端**：Axios
- **构建工具**：Vite

## 4. API接口文档

### 4.1 用户相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|----------|----------|
| `/api/login/` | POST | 用户登录 | `{"username": "...", "password": "..."}` | `{"code": 200, "message": "登录成功", "token": "..."}` |
| `/api/register/` | POST | 用户注册 | `{"username": "...", "password": "...", "email": "..."}` | `{"code": 200, "message": "注册成功"}` |
| `/api/userinfo/<userId>/` | GET | 获取用户信息 | 路径参数：userId | `{"code": 200, "data": {"id": 1, "username": "...", "email": "..."}}` |
| `/api/captcha/` | GET | 获取验证码 | 无 | 图片流 |

### 4.2 商品相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|----------|----------|
| `/apit/category/` | GET | 获取商品分类 | 无 | `{"code": 200, "data": [{"id": 1, "name": "手机"}]}` |
| `/apit/goodsbycategory/` | GET | 根据分类获取商品 | `{"category_id": 1}` | `{"code": 200, "data": [{"id": 1, "name": "小米13", "price": 3999}]}` |
| `/apit/goodsbyname/` | GET | 根据名称搜索商品 | `{"name": "小米"}` | `{"code": 200, "data": [{"id": 1, "name": "小米13", "price": 3999}]}` |

### 4.3 购物车相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|----------|----------|

## 5. 前后端交互流程

### 5.1 用户登录流程

1. 前端调用 `/api/captcha/` 获取验证码
2. 用户输入用户名、密码和验证码
3. 前端调用 `/api/login/` 接口进行登录
4. 后端验证用户信息，生成JWT token
5. 前端存储token到本地，用于后续请求的认证

### 5.2 商品浏览流程

1. 前端调用 `/apit/category/` 获取商品分类
2. 用户选择分类后，前端调用 `/apit/goodsbycategory/` 获取对应分类的商品
3. 用户可以通过搜索框调用 `/apit/goodsbyname/` 搜索商品

### 5.3 购物车操作流程

1. 前端调用 `/apit/cart/` 获取购物车列表
2. 用户可以添加商品到购物车，调用 `/apit/cart/` POST接口
3. 用户可以修改购物车商品数量，调用 `/apit/cart/<ItemId>/` PUT接口
4. 用户可以删除购物车商品，调用 `/apit/cart/<ItemId>/` DELETE接口

### 5.4 地址管理流程

1. 前端调用 `/address/` 获取地址列表
2. 用户可以添加新地址，调用 `/address/` POST接口

## 6. 核心代码解析

### 6.1 后端配置（settings.py）

```python
# 跨域配置
CORS_ALLOWED_ORIGINS = [
    'http://localhost:5173',  # 允许前端访问
]
CORS_ALLOW_CREDENTIALS = True  # 允许携带认证信息
CORS_ALLOW_HEADERS = [
    'token',  # 自定义token头
    'content-type',
]

# JWT配置
JWT_TOKEN_EXPIRE_TIME = 3600 * 24 * 24 * 24  # token有效时间
JWT_SECRET = 'abc'   # 加解密密钥
JWT_ALGORITHM = 'HS256'  # 加解密算法
```

## 7. 运行环境配置

### 7.1 后端环境

1. 安装依赖：
   ```bash
   pip install -r requirement.txt
   ```

2. 配置数据库：
   - 修改 `settings.py` 中的数据库配置
   - 创建数据库 `xiaomishopper`
   - 运行数据库迁移：
     ```bash
     python manage.py migrate
     ```

3. 启动服务：
   ```bash
   python manage.py runserver
   ```

### 7.2 前端环境

1. 安装依赖：
   ```bash
   npm install
   ```

2. 启动开发服务器：
   ```bash
   npm run dev
   ```

3. 构建生产版本：
   ```bash
   npm run build
   ```

## 8. 项目亮点

1. **前后端分离架构**：采用Vue 3 + Django REST Framework的前后端分离架构，提高开发效率和系统可维护性

2. **JWT认证**：使用JWT进行用户认证，无状态设计，便于水平扩展

3. **模块化设计**：后端采用Django应用模块化设计，前端采用组件化开发，代码结构清晰

4. **响应式布局**：前端采用响应式设计，适配不同设备屏幕

5. **完整的电商功能**：实现了用户认证、商品管理、购物车、地址管理等核心电商功能
